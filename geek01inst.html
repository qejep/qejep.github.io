Инструкции по установке здесь: Установка на Windows
Установка на Mac проще всего осуществляется с помощью Homebrew
Если приложение Homebrew у вас уже установлено, остается набрать команду
brew install todo-txt
После установки набираем в терминале команду:
cp /usr/local/opt/todo-txt/todo.cfg ~/.todo.cfg
Заходим в домашнюю директорию и ищем файл .todo.cfg
У вас должно быть включено отображение скрытых файлов (⌘+.)
В файле .todo.cfg нужно поменять настройки, указывающие путь к папкам с вашим todo.txt (если вы хотите хранить их, скажем, в облаке)

Ищем строку export TODO_DIR=$(dirname "$0") и заменяем её на строку
export TODO_DIR="/Users/YOUR_HOME_DIR/Documents"
В кавычках должен быть указан путь к папке, в которой вы намереваетесь хранить файлы todo и done

Здесь же вы можете настроить подсветку метаданных. Для этого в разделе ===Colors=== уберите хештеги перед командами export. Вы можете прописать любые цвета.

Теперь можно работать с вашими todo файлами в терминале, используя различные команды.

Еще настройки.

Чтобы не вбивать в командной строке длинное название todo.sh
прибегнем к следующей хитрости:
1. В домашней папке /Users/Ваше_имя сохраняем текстовый файл с названием .bash_profile
2. Записываем в этом файле следующую волшебную фразу:
export TODOTXT_DEFAULT_ACTION=ls
alias t='./todo.sh -d /путь/к вашему/todo.cfg'
4. Теперь набрав t вы можете сразу печатать ваши команды.

Синтаксис todo

Подробные инстукции смотрите на официальном github

Ваши задачи хранятся в обычном txt файле. Его можно редактировать прямо в блокноте или в Vim. По умолчанию каждая новая строка воспринимается todo.txt как новая задача.

Таким образом, запись
Купить слона
Помыть слона
Продать слона
Будет интерпретирована как три задачи.

Пока ничего особенного... Однако приложение todo.txt позволяет вам быстро и эффективно работать с задачами, используя особый синтаксис.
Приведем для примера строку со всеми возможными метаданными:

x (A) 2019-08-29 2019-06-01 купить слона +зоопарк @Африка due:2019-07-24

Что мы здесь имеем?
x - отметка о выполнении задания.
(A) - приоритетность задания.
2019-08-29 – дата выполнения задания.
2019-06-01 - дата создания задания.
купить слона – задание.
+зоопарк - название проекта.
@Африка - название контекста.
due:2019-07-24 - дата, на которую запланировано задание.

Синтаксис воплне читабелен и без приложения. Приложение позволяет фильтровать данные и быстро их редактировать.

Далее приведены самые важные команды

добавляем задачу
t a какая-то задача

добавляем к задачу и помечаем, что это часть определенного проекта и контекста t a какая-то задача +проект @контекст

выводим список заданий на экран
t ls

фильтруем список по заданным критериям
t ls +проект 2019-08-29 @контекст

добавляем приоритетность А заданию №1
t p 1 A

убираем приоритетность
t dp 1

помечаем задание №2 как сделанное. Оно будет автоматически перенесено в файл done.txt
t do 2

выводим на экран список контекстов
t lsc

выводим на экран список проектов
t lsprj

заменяем текст задания №2
t replace 2 некий текст

прибавляем текст к заданию №2 (если нужно что-то дописать)
t app 2 некий текст

прибавляем до задания №2 (если нужно что-то приписать сначала фразы)
t prepend 2 некий текст

переносим задание №10 в новый файл
t mv 10 имя_файла.txt

смотрим список в указанном файле
t listfile имя_файла.txt

добавляем задание в указанный файл
t addto имя_файла.txt "My bright idea"

ищем указанное слово в указанном файле
t lf имя_файла.txt носки

удаляем задание
t del 1

выводим на экран список файлов, которые хранятся в директории todo
t lf

выводим на экран список всех заданий из файла todo и done.
t lsa

Можно добавить фильтры:
+bb -bb

выводим на экран только задания с приоритетностью.
t lsp

добавляем к заданию дату создания
-t
например: t -t a купить слона

архивируем выполненные задания
t archive

отмена автоархивирования для конкретного задания
-а
