РУКОВОДСТВА

Узнать, какая версия Taskwarrior у вас установлена можно с помощью команды

task _version

Программа Taskwarrior содержит замечательное руководство на английском.

Вызвать его можно командой

man taskrc

Ещё одна полезная команда:

task help

Главная команда, которую вы должны помнить, — это вход в меню команд:

task commands

Чтобы узнать, какие отчеты по задачам вы можете получить, наберите команду

task reports

Если английский кажется вам недружелюбным, используйте данное руководство.

Да, вы можете вывести на экран логотип Программы

task logo


СОЗДАНИЕ ЗАДАЧ

Нет ничего проще, чем создать новую задачу в Taskwarrior. Просто набираем в командной строке:

task add покорить мир

Нажимаем на ввод и задача покорения мира добавлена.

Теперь можно добавить каких-нибудь дополнительных задач.

task add купить кефир

Если вам хочется большей наглядности, вы можете использовать картинки Unicode. Для этого указывайте в команде номер

task add купить U+1F95B

ПРОСМОТР ЗАДАЧ

Чтобы просмотреть список добавленных задач, наберите в командной строке:

task

На экране появится список задач. Пока он предельно прост. Как только вы начнёте добавлять к задачам дополнительные условия (дату, теги, периодичность и т.п.), Taskwarrior запустит алгоритм определения приоритетности.

Команда task будет распределять задачи в порядке их значимости. На самом деле, это сокращённый вариант команды task next.

Для просмотра списка существуют и другие команды. Например, команда

task ls

Выведет на экран список задач, рассортированных по датам.

Вы можете варьировать также и объём информации, отображающейся на экране.

Если вам нужен детальный отчёт по задачам, используйте команду

task long

Для быстрого просмотра задач больше подойдёт команда

task minimal

Наиболее оптимальной в большинстве случаев будет команда

task list

Вы получите умеренно подробный список задач. Однако с помощью команды list можно формировать самые замысловатые списки.

Хотите посмотреть задачи добавленные за последние три дня? Наберите команду

task entry.after:today-3days list

Можно посмотреть задачи добавленные вчера

task entry:yesterday list

Или пять часов назад

task entry.after:now-5hours list

Вы можете посмотреть задачи, которые засиделись в вашем списке, с помощью команды

task oldest

Если вам, наоборот, интересны только задачи, добавленные недавно, наберите команду

task newest

Если вы хотите просмотреть не только актуальные задачи, но и все задания, которые когда ли бы фигурировали в Taskwarrior, наберите команду

task all

Задачи в вашем списке будут пронумерованы. Этот номер (в левом столбце) является ID задачи. При различных манипуляциях с задачами вам нужно будет указывать их номер.

Например, если вы хотите получить детальный отчет по какой-то отдельной задаче (например задаче №1), наберите

task 1

Это краткий аналог команды task 1 information

УДАЛЕНИЕ ЗАДАЧ

Предположим, вы передумаете с покупкой кефира. Тогда задачу можно будет удалить. Посмотрите номер задачи в списке (у вас две задачи, и кефир должен быть под №2). Наберите команду

task 2 delete

Задача №2 будет удалена.

Не бойтесь, любые действия можно отменить командой

task undo

Это отменит предыдущее изменение. Если вам нужно отмотать изменения на пять шагов назад, просто используйте команду пять раз (для этого используйте повторение команды в терминале — стрелку вверх и ввод).

Посмотреть задачи, которые были удалены, вы можете с помощью команды

task +DELETED all

ЗАВЕРШЕНИЕ ЗАДАЧ

Если вы купили кефир, наберите команду

task 2 done

Команда перенесет задачу в список выполненных.

Вы можете внести какое-то дело в список выполненных сразу (вы завершили его еще до того, как успели сделать запись в Taskwarrior). Это может быть важно для последующего отчёта.

Для этого используйте команду

task log совершить подвиг

Если вы хотите посмотреть список выполненных задач, наберите команду

task completed

Но вы вольны создать более специфический запрос.

Посмотрите, чего вы добились на прошлой неделе

task end.after:today-1wk completed

Или в определённый период времени

task end.after:2019-05-01 and end.before:2019-05-31 completed

РЕДАКТИРОВАНИЕ ЗАДАЧ

Вы можете редактировать данные любой задачи, вызвав текстовый редактор:

task 1 edit

Если у вас установлен Vim, задача откроется в этом редакторе, и вы сможете внести правки.

Если вы хотите просто добавить что-то в начало или в конец задания, напишите команды

task 1 prepend Ненасильственно
task 1 append науки

Первая команда добавит "Ненасильственно" в начало задания, а вторая команда добавит "науки" в конец.

Теперь задача выглядит так:
Ненасильственно покорить мир науки.

Иногда проще дублировать уже имеющееся задание и чуть подправить его, чем печатать новое. Команда

task 10 dup

Продублирует задачу №10

Предположим, эта задача звучала так: совершить подвиг

Вы можете внести коррективы сразу при дублировании. Использовав команду

task 10 dup /подвиг/прогулку/

вы клонируете задание, но при этом слово «подвиг» будет заменено на слово «прогулку».

На самом деле вы можете использовать такую автозамену не только при дублировании.

task 1 2 3 4 /слон/бегемот/

заменит слово «слон» на слово «бегемот» во всех указанных задачах.

Если в описании задачи есть слово с опечаткой, и это слово повторяется несколько раз, используйте команду

task 3 /мера/мира/g

К любой задаче вы можете добавить краткую аннотацию.

task 1 annotate начать с Академии наук.

Лучше, если это будет короткий текст в одно-два предложения.

Если вы вызовете список (￼task list), то увидите рядом с задачей единицу в квадратных скобках. Это указание, что к задаче прикреплена одна запись.

Как вы понимаете можно прикреплять любое количество записей.

Посмотреть список задач с аннотациями можно с помощью команды

task +ANNOTATED

Убрать аннотацию можно командой

task 1 denotate

Если записей несколько, а вы хотите удалить лишь одну, укажите какие-то слова, которые в ней содержатся.

task 1 denotate наук

ТЕГИ

При работе с задачами очень эффективно использование тегов (с их помощью можно фильтровать задачи). Добавить теги можно командой

task 1 modify +статья +ВАК

Теперь к заданию №1 добавлено два тега.

Конечно, вы можете добавлять теги и сразу при создании задачи.

Удалить тег можно командой

task 1 modify -ВАК

В Taskwarrior есть особый тег +next, который выводит задачу на верхние позиции в списке.

Вы можете фильтровать задания по тегам.

Для начала выясните, какие теги уже есть в ваших списках

task tags

Теперь можно запрашивать задания с определенными тегами.

task tag:семья list

Команда выведет на экран задачи, помеченные тегом семья.

Сходного результата вы достигните командой

task +семья list

Разумеется вы можете комбинировать критерии выдачи.

Команда

task +семья and +дети list

выведет на экран задачи в которых встречаются оба тега.

Вы можете вывести на экран задачи, которые не содержат определенного тега. Для этого используется знак минуса:

task -учеба list

Комбинируя плюс, минус и логические операторы and, or, xor, вы можете создать любые фильтры.

Вы можете вывести на экран все задачи с какими-либо тегами. Для этого используйте команду

task tags.any: list

или

task +TAGGED

Вы можете просмотреть список задач, у которых нет тега, с помощью команды

task tags.none: list

ПРИОРИТЕТНОСТЬ ЗАДАЧ

Задания можно рассортировать по степени приоритетности.

task 1 modify priority:H

Теперь задание №1 имеет высшую степень важности.

В приложении доступны три уровня
H – высокий.
M – средний.
L – низкий.


Удалить приоритетность можно командой

task 1 modify priority:

У Taskwarrior есть также внутренний алгоритм высчитывания приоритетности. Например, задачи с определенной датой считаются более важными, чем задачи без даты. Задачи с аннотацией, важнее, чем задачи без аннотации и т.п.

Посмотреть список всех задач с какой-либо приоритетностью вы можете с помощью команды

task +PRIORITY

Алгоритм определения приоритетности
У Taskwarrior замысловатый метод определения важности заданий. Вы можете указать дополнительные правила, по которым будет назначаться важность какого-либо дела.

Например, команда

task config urgency.user.tag.нобелевка.coefficient 4.5

укажет Taskwarrior, что тег «нобелевка» повышает значимость задачи на 4.5 баллов.

Если вы наберете task next, то увидите, что задачи с этим тегом переместились наверх списка.

Вы можете понизить приоритетность какой-нибудь задачи, настроив другой тег.

task config urgency.user.tag.когда-нибудь.coefficient -6.0

Теперь задачи с тегом «когда-нибудь» окажутся внизу списка, потеряв шесть баллов.

Вы также можете большую значимость какому-либо проекту.

Например, команда

task config urgency.user.project.семья.coefficient 2.9

Добавить дополнительные 2.9 балла задачам в проекте семья.

ОЧЕРЁДНОСТЬ ЗАДАЧ

Чаще всего задачи связаны друг с другом. Например, задача "повесить картину", будет связана с задачей "купить гвозди".

Команда task 1 modify dep:2

Указывает, что задание №1 зависит от задания №2. Задание №2 автоматически получает более высокую приоритетность и место в списке повыше.

При этом задача №1 оказывается заблокированной. При использовании команды task next, она будет исключена из списка.

Если вы наберете команду task list, то увидите задачи с зависимостями выделены особым цветом.

Просмотреть список заблокированных задач можно с помощью команды

task blocked

Список блокирующих задач вызывается командой

task blocking

Несколько отличный отчет вы получите при использовании виртуальных тегов

task +BLOCKED
task +BLOCKING

Заметьте, что если задаче №1 присвоена важность Н, но при этом она зависит от выполнения задачи №8, последняя автоматически получит более высокую приоритетность.

Вы можете указать сразу несколько зависимостей для задачи

task 1 dep:2,3,4

Убрать зависимость можно командой

taks 1 dep:-2

ПРОЕКТЫ

Вы можете распределить задания по разным проектам. Однако помните, что задача может принадлежать только к одному проекту.

Команда task 1 modify project:наука

Отправит задание №1 в соотвествующий проект.

Дальше задания можно будет сортировать по проектам.

Поскольку назначать проект к каждому заданию дело муторное, вы можете сделать это скопом.

task 2 4 6 modify project:Нобелевка

Данная команда назначит трем заданиям место в указанном проекте.

Если вы захотите переименовать проект, наберите команду

task project:Нобелевка modify project:Шнобелевка

Вы можете создать проекты с иерархической структурой.

task add project:конференции.статья сдать статью
task add project:конференция.доклад написать выступление

Проект «конференция» содержит подпроекты «статья» и «доклад».

Вы сможете с помощью разных команд просмотреть все задачи проекта «конференция» или только задачи определенного подпроекта.

Как это сделать?

Вы можете посмотреть список имеющихся у вас проектов:

task projects

Заметьте, что если вы наберете

task project

Результат будет иным. На экран будут выведены задачи, включенные в разные проекты. При этом рассортированы они будут по степени важности. Задачи, не включённые в проекты, отображаться не будут.

Чтобы увидеть список задач из разных проектов, рассортированных в соотвествии с календарными датами, наберите:

task project list

Вы можете вывести на экран задачи, относящиеся к определенному проекту:

task project:Нобелевка list

Можете вывести задачи нескольких проектов:

task ‘(project:Нобелевка or project:Похудение)’ list

Естественно, вы можете комбинировать поисковые критерии.

Например, вывести на экран задачи, относящиеся к проекту «Нобелевка» и созданные на прошлой неделе:

task project:Нобелевка and entry.after:today-1wk list

ЗАДАЧИ, ПРИВЯЗАННЫЕ К ДАТЕ

Следующий важный шаг — определиться с датами.

Прежде всего, существует несколько полезных команд, позволяющих вам сориентироваться во времени

task calendar

Выводит на экран календарь на несколько месяцев вперёд (если ваш терминал развернут на полный экран, вы увидите календарь на полгода).

Мы вернемся к функциям календаря в конце этого раздела. Сначала нужно внести в Taskwarrior задачи, привязанные к дате.

Представьте, что вам нужно узнать какая дата будет через 21 день. Наберите команду

task calc now + 21d

Вы можете использовать специальные сокращения, чтобы узнать, например, дату окончания квартала

task calc eoq

Если дата для определенной задачи известна заранее, вы можете указать ее сразу при заполнении

task add помыть слона due:2019-06-20

Вы также можете добавить дату к уже существующей задаче

task 1 modify due:2019-06-21

Вы также можете дублировать дату из другой задачи

task add купить шампунь due:1.due

Если вы теперь запросите отчет task list, то увидите, что задачи рассортированы по датам. Ближайшие по времени будут находиться наверху списка.

Вы можете просмотреть список задач с датами, набрав команду

task due.any: list

Можно, наоборот, посмотреть задачи без дат

task due.none: list

Ещё одна полезная команда

task _get 1.due.week

Вы сможете узнать на какую неделю года назначена задача.

Убрать дату из задачи можно командой

task 1 modify due:

Taskwarrior просто сотрет существующую дату.

Даты и время можно указывать по-разному.

2020-03-13T12:45
Будет означать, что задача запланирована на 13 марта 2020 года, на 12:45

Если вы не указали время (это не обязательный элемент), по умолчанию будет считаться, что задача назначена на самое начало дня — 00:00:00.

Если вы указали только время due:14:30 будет считаться, что речь про полвторого сегодняшнего дня.

Вы можете указать порядковый номер дня в месяце. Команда

task add продать слона due:11th

назначит задаче ближайшее одиннадцатое число.

Можно указать на какой день по счету в году запланирована задача (следует указывать трехзначное число, поэтому пятое января будет выглядеть так – 005)

Команда

task add купить слона due:2020-073Т12:30

Назначит задачу на 13 марта 2020 года на полпервого дня.

Вы можете указать номер недели и номер дня на неделе

task add купить слона due:2020-W11-5Т12:30

Номер недели нужно указывать как двухзначное число, поэтому вторая неделя января будет выглядеть как W02.

Если вы не указали день недели или время

task add купить слона due:2020-W11

будет считаться, что подразумевалось воскресенье, 00:00:00 (В США отсчет недели идет с воскресенья).

Это может слегка запутать. В настройках можно указать, что первым днем недели следует считать понедельник.
Для этого напишите в файле .taskrc функцию weekstart=monday

Теперь указав номер дня недели, вы будете уверены, что цифра 2 – это вторник. Тем не менее, если день не указан, Taskwarrior все равно будет вести подсчеты по американскому образцу.
Поэтому команды due:2020-W11 и due:2020-W11-1, при европейских настройках будут отличаться.

Кроме цифрового формата введения дат, вы можете воспользоваться буквенными сокращениями.

due:now — текущее локальное время и дата.
due:today — текущая дата, время 00:00:00.
due:sod — начало следующего дня, время 00:00:00.
due:tomorrow — тоже, что и предыдущая команда.
due:eod — текущая дата, время 23:59:59
due:monday — ближайший понедельник, время 00:00:00.
due:january — первое число указанного месяца, время 00:00:00
due:someday — 2038-01-18, время 00:00:00.
due:soy — 1 января следующего года, время 00:00:00.
due:eoy — 31 декабря текущего года, время 00:00:00.
due:soq – начало следующего квартала (Январь, Апрель, Июль, Октябрь), первое число, время 00:00:00
due:eoq – конец текущего квартала (Март, Июнь, Сентябрь, Декабрь), последний день месяца, время 23:59:59
due:som – Первое число следующего месяца, время 00:00:00
due:eom – Последнее число текущего месяца, время 23:59:59
due:sow – ближайшее воскресенье, время 00:00:00
due:eow – ближайшая суббота, время 00:00:00
due:soww – ближайший понедельник, время 00:00:00
due:eoww – ближайшая пятница, время 23:59:59
due:2nd – прибавление количества дней к текущей дате, время 00:00:00

Вы также можете указать не конкретную дату, а время через которое планируется приступить к задаче.

Например, команда

task add преодолеть себя due:5d

Будет означать, что преодоление себя планируется через пять дней после сегодняшнего дня.

Обозначения здесь следующие:

due:3h – через три часа.
due:3d – через три дня.
due:3w – через три недели.
due:wk – через неделю.
due:3m – через три месяца.
due:mo – через месяц.
due:1q – через 90 дней.
due:1y – через год.

Если вы хотите посмотреть просроченные задания (а такие у вас точно будут), наберите команду

task overdue

Теперь стоит вернуться к функциям календаря. Он позволяет посмотреть на ваши планы с высоты птичьего полета.

Если в Taskwarrior уже есть датированные задания, то набрав команду

task calendar

Вы увидите не просто даты, а еще и удобные цветовые обозначения (задачи, просроченные задачи).

Можно вывести на экран календарь на целый год

task calendar 2020

Вы, разумеется, можете посмотреть любой год. Это удобная функция. Например, если какие-то задачи зависли с прошлого года, вы сразу заметите их в календаре.

Вы также можете вывести на экран несколько месяцев, начав с какого-то определенного. Например, команда

task calendar 7 2019

Покажет несколько месяцев (сколько поместится в один ряд), начиная с июля 2019.

Команда

task calendar due

Выведет на экран несколько месяцев (сколько поместится в одном ряду), начиная с самого раннего запланированного задания.

ФИЛЬТРАЦИЯ ДАТ

Хотите просмотреть только задачи на сегодня? Используйте команду

task +TODAY

Того же результата вы добьётесь командой

task due:today list

Команда task +TOMORROW позволит вам просмотреть список задач назавтра.

Вы можете также просмотреть задачи ближайших дней

task +DUE

...текущей недели

task +WEEK

или месяца

task +MONTH

или квартала

task +QUARTER

Вы даже можете вывести на экран задачи датированные текущим годом

task +YEAR

Можно посмотреть просроченные задачи

task +OVERDUE

Вы также можете комбинировать разные критерии. Например, команда

task +DUE -TODAY

выведет на экран команды ближайших дней, исключая сегодняшний.

ДОПОЛНИТЕЛЬНЫЕ УСЛОВИЯ ДЛЯ ЗАДАЧ

Функция due назначает крайний срок выполнения задачи. Но что если вы хотите обозначить не только финиш, но и старт? Допустим, к выполнению определенного дела не имеет смысла приступать раньше определенного срока.

Для этого существует функция scheduled.

task add вытащить бегемота из болота due:2020-12-08 scheduled:2020-11-08

Как только наступит 8 ноября 2020, задача «вытащить бегемота» обретёт статус ready.

Это означает, что можно приступать к спасательной операции.

Вы можете посмотреть список готовых задач, используя команду

task ready

Или фильтр

task +READY

Если функция scheduled не задана, Taskwarrior рассматривает задачи, как готовые к выполнению. Тогда в вашем списке будут все датированные задачи.

В общем, scheduled удобный способ отсечь задачи, которые несвоевременны.

Вы можете посмотреть список задач, старт которых назначен на определенный срок, использовав виртуальный тег:

task +SCHEDULED

До старта некоторых задач могут быть месяцы. Чтобы они не мозолили глаз и не увеличивали стресс, можно их запрятать до времени. Для этого существует функция wait.

task add вытащить из болота бегемота due:2020-12-08 scheduled:2020-11-08 wait:october

До начала октября задача будет исключена из списков. Вы, конечно, можете указать и какую-то конкретную дату.

Если вам вдруг захочется посмотреть ждущие своего часа задачи, используйте команду

task waiting

Есть задачи, которые становятся неактуальными после дедлайна. Бессмысленно держать их в списке. Можно, конечно, удалять их вручную. Но для этого существует специальная функция until.

task add вытащить из болота бегемота due:2020-12-08 scheduled:2020-11-08 wait:october until:2020-12-15

Предположим, что после 15 декабря тащить бегемота будет уже поздно. Taskwarrior просто удалит задачу из списка.

Вам необязательно прописывать все даты в задаче. Иногда для этого удобнее использовать специальные выражения

task add вытащить из болота бегемота due:2020-12-08 scheduled:due-4d wait:due-7d until:due+7d

Список задач, которые самоунитожатся, можно вывести на экран командой

task +UNTIL

ТЕКУЩИЕ ЗАДАЧИ

Если у вас есть задачи, которые не выполнить за один присест, вы можете использовать специальную команду

task 1 start

Задача будет выделена особым цветом и займет верхнее место в списке. Вы сразу заметите, что начало положено, но процесс не завершён.

Это очень полезно, чтобы не потерять след. Видно над какими проектами вы трудитесь.

Список активных (стартовавших) заданий можно получить командой task active

Остановить задачу можно командой

task 1 stop

КОНТЕКСТЫ

Некоторые задачи привязаны к определенному месту или условиям. Например, их можно сделать только на работе или дома. Задачи, которые вы сейчас физически не можете выполнить, не должны мешаться и нервировать.
Функция «контексты» позволяет вывести на экран список задач, осуществимых только при данных условиях.
Конечно, тоже самое можно сделать фильтруя задачи по тегам. Контексты просто автоматизируют процесс. Вы создаете правило, которое будет применяться к задачам из вашего большого списка.

Создадим контекст для учёбы в университете

task context define учёба +эссе or +доклад or +курсовая

Теперь набрав команду

task context учёба

вы включите контекст. Задачи, которые не содержат тегов +эссе или +доклад, или +курсовая, будут жестоко игнорироваться.

Команда task list выведет на экран только задачи из контекста «учёба».

Вы можете комбинировать любые условия для создания контекста (теги и проекты). Например, команда

task context define выходные +семья or pro:отдых -врач

создаст контекст, в котором вы увидите задачи с тегом «семья» или с проектом «отдых» за исключением задач с тегом «врач».

Будьте внимательны при использовании логических операторов.

Посмотреть, какой контекст включен, можно с помощью команды

task context show

Отключить контекст можно командой

taks context none

Вы можете создать множество разных контекстов под разные задачи. Чтобы посмотреть список контекстов наберите

task context list

или

task _context

Если вам нужно переключиться на другой контекст, просто укажите его в команде

task context имя_контекста

Вы можете удалить любой контекст, потерявший актуальность, командой

task context delete имя_контекста

Задания, входившие в контекст, при этом не удалятся, а просто утратят атрибут.

ОТЧЕТЫ ПО ЗАДАЧАМ

Taskwarrior позволяет формировать самые разнообразные отчёты. Вы можете быстро просмотреть, как обстоят дела с различными задачами, получить необходимые данные.

Команда

task stats

Выведет на экран сводную таблицу. Вы узнаете, сколько задач выполнено, сколько удалено, сколько проектов существует и т.п.

Возможно, вам не нужна подробная таблица. Вы можете сделать запрос по какому-то отдельному аспекту.

Например, посмотреть сколько всего задач в ваших списках.

Команда

task count

выведет на экран цифру всех известных Taskwarrior задач (как невыполненных, так и выполненных).

Это не самый информативный отчёт. Функция count демонстрирует только число задач.

Вы можете задать любые критерии подсчета. Например, команда

task status:pending count

покажет количество нерешенных задач в вашем списке. Вы также можете использовать специальный тег:

task +PENDING count

Команда

task project:диссертация count

Покажет количество задач в проекте «диссертация» (конечно, если у вас есть такой проект).

Команда

task +семья count

покажет количество задач с тегом «семья», а команда

task -семья count

покажет число задач без этого тега.

task ghistory выводит на экран графический отчет о заданиях, которые были добавлены, выполнены или удалены за месяц.

task history выдаст вам ту же информацию, но в форме таблицы.

Команда

task timesheet

продемонстрирует какие задания выполнены за неделю.

Вы можете сузить поисковую выдачу по времени. Команда

task ghistory.annual

сформирует годовой отчёт.

Команда

task ghistory.monthly

покажет график ваших достижений за месяц.

То же будет справедливо для команд

task history.annual
task history.monthly

Другие отчёты в виде графиков можно получить с помощью команд

task burndown.daily
task burndown.weekly
task burndown.monthly
task burndown.yearly

Здесь вы увидите соотношение выполненных, отменённых и невыполненных задач. Чтобы графики отображались корректно, лучше перейти в полноэкранный режим.

Команда

task summary

сформирует графический отчет, в котором задачи будут распределены по проектам (задачи без проекта окажутся в отдельной группе).

ПОИСК ПО ТЕКСТУ

Что если вы хотите найти определенную задачу в длинном списке? Вы помните, что она содержит слово «синхрофазатрон».

Если так, то поисковых запрос будет выглядеть так:

task /синхрофазатрон/ list

Поиск можно осуществлять и по части слова, так что /синхр/ тоже подойдет.

Taskwarrior осуществит поиск, как в описаниях задач, так и в прикрепленных примечаниях (заметьте, что названия проектов и теги в результаты поиска включены не будут).

Используйте функцию long вместо list, если вам нужно посмотреть полный текст примечаний.

Если вы хотите найти целую фразу, а не отдельное слово, используйте одиночные кавычки

task '/купить синхрофазатрон/' list

Вы можете даже использовать в поиске регулярные выражения (хотя они могут пригодится только при очень длинных списках задач).

Если вы хотите осуществить поиск и в тексте примечаний и в названиях задач, наберите команду

task синхрофазотрон list

ВИРТУАЛЬНЫЕ ТЕГИ

Список всех специальных тегов можно просмотреть, набрав команду

task _tags

Чтобы проверить какие виртуальные теги прикреплены к конкретной задачей, наберите команду

task 1 info

Вас будет интересовать раздел Virtual tags.

ПОВТОРЯЮЩИЕСЯ ЗАДАЧИ

Каждый год следует посещать зубного, каждый месяц приходится платить за квартиру, каждую поливать цветы и т.п. В нашей жизни много цикличных задач.
Конечно, удобно, когда при выполнении подобных задач, программа сама добавляет их в список с учетом интервала.
Taskwarrior позволяет это делать.

Команда

task add наверстать упущенное за год due:2019-12-31T09:00 recur:yearly

Добавит задачу с годичным циклом. Заметьте, что для повторяющихся задач необходимо указывать дату начала (время опционально).

Taskwarrior имеет следующие опции:

Ежедневное повторение

recur:daily

или

recur:1da

Можно задать повторение через какой-то интервал. Например, через три дня

recur:3da

Вы можете указать десять дней или семь дней. Однако для последнего случая существует отдельная команда.

recur:weekly

Вы можете указать повторение задачи через определённое количество недель. Например, три

recur:3wks

Можно использовать такие выражения как

recur:biweekly или recurve:fortnight

Для заданий, повторяющихся каждые две недели.

Вы можете настроить повторение задачи только в рабочие дни с отдыхом по выходным.

recur:weekdays

Вы можете повторять задачу ежемесячно

recur:monthly

или через определённое количество месяцев

recur:2mo

Если задача повторяется раз в квартал, вы вольны указать команду

recur:quarterly

или отсчитать нужное количество кварталов

recur:2qtrs

Поскольку, два квартала это шесть месяцев, вы могли бы написать и

recur:semiannual

Вы можете повторять задачу ежегодно с помощью команды

recur:annuals

Или

recur:yearly

Можете использовать интервал в два года (или любой другой интервал)

recur:2yrs

В последнем случае вы можете использовать выражение

recur:biyearly

Но что если после определенного количества повторов задачу следует снять?

Для этого добавьте в команду выражение until:

task add Pay rent due:28th recur:monthly until:now+1yr

Вы можете указать конкретную дату, например, until:2019-12-31

Посмотреть список всех повторяющихся задач вы можете с помощью команды

task recurring
